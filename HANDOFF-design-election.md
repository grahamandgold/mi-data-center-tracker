# Design Handoff — "Your Voice 2026" election page

**For:** a design-focused Claude (visual/style rework)
**Goal:** rework the look & feel. **Content, data, and copy are correct and verified — this is a styling job. Do not change data or numbers.**

---

## 1. Where it lives
- **File:** `your-voice-2026.html` (repo root of `github.com/grahamandgold/mi-data-center-tracker`)
- **Live:** https://grahamandgold.github.io/mi-data-center-tracker/your-voice-2026.html
- **Runtime:** `support.js` (do NOT edit), plus data files it fetches: `map-data.json`, `geo/mi-house-districts.geojson`, `geo/mi-senate-districts.geojson`, candidate photos in `img/candidates/`.

## 2. READ THIS FIRST — it's not plain HTML
This page is a **custom `<x-dc>` component** hydrated by `support.js` (a small React-based runtime). The candidate data + all rendering live in an inline `<script type="text/x-dc" data-dc-script>` containing `class Component extends DCLogic { ... }`. Cards, the bracket, the dashboard, and the map are **generated by JS** (string templates) into container `<div>`s.

**Almost every style is an inline `style="..."` attribute.** There is no external stylesheet and very few CSS classes. Hand-editing hundreds of inline styles is how you break things — see §5.

## 3. The safe way to restyle (do this)
The whole palette/typography runs off **CSS variables** defined in the `<head>` `<style>`. Change the look by editing these tokens + adding global rules, NOT by touching inline styles one by one.

```
--navy:#0C1E33;  --ink:#14212E;   --mist:#EDF1F6;  --paper:#FFFFFF;
--panel:#F5F8FC; --panel-2:#E7EEF6; --lake:#1E6FC4 (primary blue);
--soft:#4C5661;  --faint:#5F6875;  --hair:rgba(12,30,51,0.13);
--cherry:#C4342C (Republican red / accents); --copper:#C9781F (highlights);
--pine:#1E8E52 (on-record green);
```
Fonts (Google Fonts, already linked): **Archivo** (400–900) for display/body, **IBM Plex Mono** (400–600) for labels/eyebrows.

To go further, add a real `<style>` block and attach classes — inline styles win, so you'll need `!important` or higher-specificity selectors to override them. Prefer restyling via tokens + a stylesheet over rewriting inline styles.

## 5. Hard rules — do NOT break
- **Keep these element IDs** (JS writes into them): `#dash-donut`, `#dash-bars`, `#bracket`, `#cards`, `#dmap`, `#cand-count`, `#onrec-top`, `#zip`, `#zip-note`, and card IDs `#cand-<slug>`.
- **Don't edit the `<script type="text/x-dc">` logic** unless a style truly requires it. Prefer CSS.
- **After ANY structural edit, load the page and check the browser console.** A thrown error in `componentDidMount` used to blank the entire page. (There's now per-section `try/catch`, but a broken section will still silently disappear — the console tells you.)
- **Don't touch** `support.js`, `map-data.json`, the `geo/` files, or `scripts/`.
- Keep the brand tokens + Archivo / IBM Plex Mono unless the redesign intentionally changes them.

## 6. See it / deploy it
- Preview locally: `cd` to repo, `python3 -m http.server 8000`, open `http://localhost:8000/your-voice-2026.html`.
- Deploy (GitHub Pages serves `main`): `git add your-voice-2026.html && git commit -m "…" && git pull --rebase --autostash origin main && git push origin main`. CDN caches ~1–2 min; add `?v=N` to bust.
